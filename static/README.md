3D трекинг - (пока) статика.
На вход нужны:
- предикты SAM3; (добываются в inference_on_video в папке sam)
- также эмбеддинги кадра от SAM3 (там же)
- глубины, интринсики и экстринсики от DA3;

Для каждого трека сохраняется его предсказанный класс и маски для каждого кадра (где объект есть).

Загрузка всех данных и вызов трекера в 3d_tracker.ipynb.

Класс Track и класс Tracker находятся в файле tracker.py
Также там находятся VoxelMap для хранения вокселей трека и EMAFilter для обновления эмбеддингов.

Основные особенности трекера:
- Используемая метрика w_ioa*ioa+w_dist*dist_similarity+w_emb*emb_similarity
То есть, это аналог ioa для пойнтклаудов, расстояние между центрами (нормировка экспонентой), и косинусная близость эмбеддингов.
Все метрики находятся в metric_utils.py.
Важно - при расчете ioa сигма (грубо говоря, это расстояние между точками, при котором они считаются совпадающими), выбирается адаптивно от размера объекта (например для маленькой статуэтки 5 см это критично, а для большого дивана - вообще ничего).
- Правила обновления - задаются классом SmartUpdateRules в tracker.py. Грубо говоря, для обновления эмбеддингов, объект должен быть хорошо виден (хорошее пересечения трека и детекции или детекция больше трека), для обновления вокселей порог пониже, а если пересечение совсем низкое - значит объект почему-то "сдвинулся", в таком случае сбрасываем геометрию.
- Порядок рассмотрения треков: active SAM predict -> active best track -> lost best track -> tentative SAM predict  -> tentative best track. Если для детекции не нашлось совпадений ни на одном этапе - создается новый трек. Все новые треки имеют статус tentative, пока не будут подтверждены min_hits.

Скрипты для работы с данными:
-  data.py - извлечение визуальных и текстовых эмбеддингов, а так же пойнтклауда объекта по маске. Важно уточнить, что при извлечении к маске применяется эрозия, чтобы не учитывать шумные края. Также при извлечении объекта используется фильтрация выбросов по ближайшим соседям.
-  mask_utils.py - инструменты для работы с масками, эрозия и разделитель/мерджер масок. Так как SAM может предсказать два объекта в одной маске, используется постпроцессинг с разделением через DBSCAN. Важно - для ускорения используется ресайз до низкого разрешения (~300 пикселей). Маски, размером меньше порога добавляются к самой большой маске, которая остается на том же треке, для остальных больших масок создаются новые треки.
-  point_utils.py - получение пойнтклаудов из карт глубины и параметров камеры. Также используется фильтрация по conf DA3 и статистическая по ближайшим соседям.

Скрипт для визуализации (2d) - track_vis_utils.py. Отрисовка масок поверх кадров или номеров треков (set of marks).
